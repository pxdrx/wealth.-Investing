# Mockups Primeiro

## Lição

> **Apresentar HTML mockups antes de implementar mudanças de UI.**

## Processo

1. Criar mockup HTML standalone
2. Screenshot ou servir local
3. Apresentar para aprovação
4. Só então implementar no codebase

## Por quê

- Feedback visual antes do código
- Iteração rápida sem refactoring
- Alinhamento de expectativas
- Evita "não era isso que eu queria"

## Exemplo

Macro Intelligence teve 2 versões de mockup:
- `mockup-macro-intelligence.html` (V1 — chosen)
- `mockup-macro-intelligence-v2.html` (V2 — reference)
